HTML

HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content.

Sematic Tag | Inline Element | Block Level Element | Image Tag | Anchor Tag

Semantic Tags

A semantic tag in HTML is an element that clearly describes its meaning to both the browser and the developer. Semantic tags make your code more readable and improve accessibility and SEO.

Examples of Semantic Tags

Tags Explanation
Header Tag The header tag in HTML element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
Article Tag The article tag in HTML element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable
Section Tag The section tag in HTML element represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. Sections should always have a heading, with very few exceptions.
Nav Tag The nav HTML element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.
Aside Tag The aside HTML element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
Main Tag The main tag in HTML element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
Footer Tag The footer tag in HTML element represents a footer for its nearest ancestor sectioning content or sectioning root element. A footer tag is the typically contains information about the author of the section, copyright data or links to related documents.

Inline level Elements

  • A block-level element always starts on a new line.
  • It occupies the entire horizontal space of its parent element
  • Vertical space equal to the height of its contents, thereby creating a "block".

    • Example:

    1. Structural elements:<header>, <footer>, <nav>, <section>, <article>, <aside>, <main> >div>.
    2. Text content elements: <p>, <h1>to <h6>.
    3. List elements:<ul>, <ol>, <li>, <dl>, <dt>, <dd>.
    4. And some tags: <form>, <hr>, <table>.

Block level Elements

  1. Inline elements in HTML are elements that do not begin on a new line and only occupy the width necessary for their content.
  2. They are designed to flow within the existing line of text, allowing other elements to appear on the same line if there is sufficient space.
  3. It always start on a new line and take up the full available width.

    4. Example:

    • <a>: Defines a hyperlink.
    • <span>: A generic inline container for applying styles or scripting to a specific part of text.
    • <strong> and <em>: Used for strong emphasis and emphasized text, respectively.
    • <img>Embeds an image.
    • <input>, <label>, <button>:Common form elements.

Image Tag

The <img> HTML element embeds an image into the document.

Attributes of Img Tag

  • The src attribute holds the path to the image you want to embed.
  • Alt attribute is used to displayed text on the page if the image can't be loaded for some reason.
    for example, network errors, content blocking, or link rot.
  • we can use both width and height to set the size of the image
